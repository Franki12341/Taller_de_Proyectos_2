# Matriz de trazabilidad de SmartSched-UC

## Propósito

Relacionar los requisitos funcionales y no funcionales de SmartSched-UC con sus actores, módulos, archivos de implementación, pruebas, resultados, evidencias, riesgos, incidentes, defectos, cambios y registros de control de versiones.

Esta matriz permite comprobar que cada capacidad relevante del MVP cuenta con implementación verificable y evidencia suficiente para su revisión académica.

## Criterios de estado

| Estado                            | Significado                                                                             |
| --------------------------------- | --------------------------------------------------------------------------------------- |
| Completado                        | El requisito fue implementado y cuenta con evidencia verificable                        |
| Completado con mejora recomendada | El requisito fue implementado, pero conserva oportunidades de mejora                    |
| En tratamiento                    | El requisito cuenta con implementación o evaluación, pero presenta hallazgos pendientes |
| Parcialmente validado             | Existe evidencia parcial, pero no una validación integral                               |
| No aplica                         | El elemento no corresponde al requisito evaluado                                        |

## Hallazgos que permanecen en tratamiento

Aunque el núcleo funcional está completado, continúan las siguientes acciones de mejora:

1. reducir las vulnerabilidades del backend;
2. priorizar las vulnerabilidades críticas y altas del frontend;
3. incrementar la cobertura del frontend;
4. ejecutar una auditoría integral de accesibilidad WCAG;
5. reparar completamente los hooks de Husky en Windows;
6. continuar modularizando `client/src/App.js`;
7. verificar la codificación UTF-8 en todos los documentos históricos.
